# TCP 通信协议

## 帧格式

```text
+--------+--------+------+--------+----------+----------+
| 0xA5   | 0x5A   | Type | Length | Payload  | Checksum |
+--------+--------+------+--------+----------+----------+
```

- `Type`：1 字节
- `Length`：1 字节，表示负载长度，范围 0 到 255
- `Checksum`：从 `0xA5` 开始到最后一个 Payload 字节为止，所有字节按位异或（XOR）

TCP 是字节流协议，因此 ESP32 解析器可以处理以下情况：

- 一个完整帧被拆分到多个 TCP 包中
- 一个 TCP 包中包含多个完整帧

## 消息类型(Type字段)

```text
0x01 键盘报告（Keyboard report）
0x02 绝对坐标触摸板报告（Absolute touchpad report）
0x03 Ping
0x04 释放全部（Release all）
```

## 键盘报告负载（Keyboard report payload）

长度：8 字节

```text
byte0: 修饰键位（modifier bits）
byte1: 保留位，通常为 0
byte2: keycode 1 #剩下5个对应的是按下了什么按键,也就是最多按下5个
byte3: keycode 2
byte4: keycode 3
byte5: keycode 4
byte6: keycode 5
byte7: keycode 6
```

修饰键位定义：

```text
bit0 Left Ctrl
bit1 Left Shift
bit2 Left Alt
bit3 Left GUI #左windows键
bit4 Right Ctrl
bit5 Right Shift
bit6 Right Alt
bit7 Right GUI
```

常见 HID 键码：

```text
A = 0x04
B = 0x05
C = 0x06
1 = 0x1E
2 = 0x1F
3 = 0x20
Enter = 0x28
Esc = 0x29
Backspace = 0x2A
Tab = 0x2B
Space = 0x2C
```

要释放所有键盘按键，可以发送一个 8 字节全 0 的键盘报告，或者发送 `Release all` 命令。

## 绝对坐标触摸板报告负载（Absolute touchpad report payload）

长度：6 字节

```text
byte0: 按钮位（buttons）
byte1: x 低字节
byte2: x 高字节
byte3: y 低字节
byte4: y 高字节
byte5: contact
```

- X 和 Y 为无符号小端（little-endian）16 位整数
- 有效坐标范围为 0 到 32767
- `contact = 1`：手指接触
- `contact = 0`：手指离开

按钮位定义：

```text
bit0 左键
bit1 右键
bit2 中键
```

## 键盘帧示例

按下 `A` 键：

```text
Payload: 00 00 04 00 00 00 00 00
Frame:   A5 5A 01 08 00 00 04 00 00 00 00 00 F2
```

释放键盘：

```text
Payload: 00 00 00 00 00 00 00 00
Frame:   A5 5A 01 08 00 00 00 00 00 00 00 00 F6
```

## 触摸板帧示例

在中心点 `(16384, 16384)` 移动/接触，不按按钮：

```text
Payload: 00 00 40 00 40 01
```
